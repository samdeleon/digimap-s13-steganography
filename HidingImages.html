Hiding Images

Hiding a secret image within a cover image.

HOW IT WORKS
Hiding images involve 2 inputs: the cover image and the secret image. We will be using this blue photo as the cover image. The RGB Color Model was used for our project where each pixel in a photo has RGB components: red bits, green bits, blue bits. The leftmost bits or the most significant bits (short for MSB) makes up most of the pixel’s color. This means that changes to the rightmost bits or the least significant bits (short for LSB) won't make that much of an effect for the overall output. This is why the LSB method uses the LSB of the cover image to hide the MSB of the secret image. In the example below, the hidden bits number is 1. The following are the steps that take place:

  1. The cover image takes 7 MSB.
  2. The secret image takes 1 MSB.
  3. These bits are combined where the leftmost bits are the cover image’s bits, and the rightmost bits are the secret image’s bits.
REMEMBER:
More hidden bits = the image is easier to see
Less hidden bits = the image is harder to see
